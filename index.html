MongoDB 數據庫模型 (Schema) 設計

若您日後使用 Node.js + Express + Mongoose 建置 MongoDB 後端 API，可以使用以下定義的兩套數據集合 (Collections)：

// 1. 修持人與統計集合 (UserSchema)

const UserSchema = new mongoose.Schema({
  userId: { type: String, required: true, unique: true }, // 'dicky' | 'hannah'
  name: { type: String, required: true },               // 'Dicky' | 'Hannah'
  counts: {
    jingang: { type: Number, default: 0 },              // 金剛經遍數
    dizang_shang: { type: Number, default: 0 },         // 地藏經 上卷未進位遍數
    dizang_zhong: { type: Number, default: 0 },         // 地藏經 中卷未進位遍數
    dizang_xia: { type: Number, default: 0 },           // 地藏經 下卷未進位遍數
    dizang_full: { type: Number, default: 0 }           // 地藏經 已自動進位的完整部數
  },
  targets: {
    jingang: { type: Number, default: 108 },
    dizang_full: { type: Number, default: 100 }
  }
});

// 2. 修持流水紀錄集合 (RecitationLogSchema)

const RecitationLogSchema = new mongoose.Schema({
  userId: { type: String, required: true },             // 'dicky' | 'hannah'
  scriptureId: { 
    type: String, 
    enum: ['jingang', 'dizang_shang', 'dizang_zhong', 'dizang_xia'], 
    required: true 
  },
  increment: { type: Number, required: true },          // 增加遍數
  note: { type: String, default: '' },                   // 備註 (如: 早課)
  timestamp: { type: Date, default: Date.now }          // 修持時間
});

💡 本應用目前預設支援 localStorage 本地保存，資料關閉瀏覽器後依然存在。
